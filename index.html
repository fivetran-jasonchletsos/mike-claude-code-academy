HOUR 1 Training Camp: Get Claude Talking to You
iPhone or Windows +

Goal

Get Claude answering your first message today, on whichever device is already in your hand. The iPhone tab below is the fastest way in — no installs, just an app. The Windows tab is where the real building starts, and Hours 2 through 5 all continue there, since Claude Code itself needs a terminal to run. If you could hand-edit MySpace HTML back in the day, the Windows side is going to feel familiar fast.

Steps

  1. Download the Claude app from the App Store.
  2. Log in with a free account at claude.ai, or create one right in the app.
  3. Send your first message: "hi! what can you help me do?"
  4. Snap a photo of a card you own and ask, "what player and set is this, and roughly what condition does it look like from this photo?" Treat the answer as a starting point, not gospel — always double-check anything you're about to pay real money on.
  5. Try the microphone icon and dictate a note instead of typing — handy while you're sorting a box of cards on the couch.
  6. Ask it to draft something on the spot: "write a short eBay-style description for this 2018 Jason Kelce card, near-mint condition." You'll polish it later on the laptop.
  7. Keep a running list in the app of cards to look up later — you'll bring that list over to the tracker you build in the Trade Block bonus.

Checkpoint

If something breaks

The iPhone app is for chatting, photos, and voice — it can't run terminal commands or build project files the way Claude Code does on Windows. That's exactly what the Windows tab above, and Hours 2 through 5, are for.

What you'll need

  • A Windows 10 or 11 laptop with internet
  • A free account at claude.ai (sign up if you don't have one)
  • 60 minutes and a little patience for first-time installer screens

Steps

  1. Open Windows Terminal (built into Windows 11; on Windows 10, grab it free from the Microsoft Store). This is where you'll type commands.
  2. Install Node.js LTS from nodejs.org — download the installer and click through the defaults.
  3. Close and reopen Windows Terminal so it picks up the new install.
  4. Check it worked:
    node -v npm -v
    You should see version numbers, not an error.
  5. Install Claude Code itself:
    npm install -g @anthropic-ai/claude-code
  6. Make a folder to practice in and move into it:
    mkdir C:\Users\Mike\claude-practice cd C:\Users\Mike\claude-practice
  7. Launch it:
    claude
    A browser tab opens the first time — log in with your Claude account.
  8. Once you see the Claude Code prompt, type this and hit Enter:
    hi! what folder are we in, and what's inside it?

Checkpoint

If something breaks

"'claude' is not recognized" almost always means the terminal needs a restart to see the new install — close it completely and reopen it. If that doesn't fix it, restart the whole laptop once.
HOUR 2 Playbook: Folders, Files & the Permission Read-Option
Windows +

Goal

Get comfortable moving around folders in a terminal, and understand exactly what those "yes / no" prompts from Claude Code mean before you click through them.

Steps

  1. Practice moving around (in your claude-practice folder from Hour 1):
    dir (list what's in this folder) cd .. (go up one level) mkdir card-vault cd card-vault
  2. Start Claude Code again here: claude
  3. Ask it to make a file: "make a file called notes.txt that says hello world" — watch it show you the change and ask permission before it touches anything.
  4. Read the prompt every time, at least while you're learning. It tells you exactly what file it's about to create or edit — same idea as reading a permission popup before it just does something.
  5. Try the built-in help menu: type /help to see what commands Claude Code understands on its own.
  6. Ask it to explain something back to you: "what does notes.txt say right now?"

Checkpoint

If something breaks

Prompts moving too fast? You can always scroll up in Windows Terminal, or just ask Claude directly: "what did you just do?"
HOUR 3 Film Study: Prompting, Plans & the Undo Button
Windows +

Goal

Learn how to actually talk to Claude Code so it builds what's in your head, plus set up your safety net (git) so nothing is ever truly lost.

Steps

  1. Specific beats vague. Instead of "make an app," try: "build a simple script that prints 5 random Eagles players from a list."
  2. For anything bigger than a one-line fix, ask for a plan first: "make a plan for X before you build it" — review the plan, then say go.
  3. Practice it: ask Claude to plan (not build yet) the Card Vault Finder project you'll build in Hour 4, just to see what a plan looks like.
  4. Install Git for Windows from gitforwindows.org if you don't have it yet.
  5. In your project folder, save your first checkpoint:
    git init git add . git commit -m "first checkpoint"
  6. After every chunk of real progress, ask Claude: "commit this with a good message." That commit is your undo button forever.
  7. Practice saying "undo that last change" and watch how Claude handles it.

Checkpoint

If something breaks

"'git' is not recognized" → reinstall Git for Windows, then fully close and reopen the terminal.
HOUR 4 Two-Minute Drill: Build the Iggles Card Vault Finder
Windows +

Goal

Build a real, working tool that searches a list of Eagles legends and modern-era stars — the same idea behind the "Play" section of this very site.

The project brief — paste this into Claude Code almost word for word

Build me a small command-line tool called Card Vault Finder. It should have a list of at least 15 Philadelphia Eagles legends and 15 modern-era Eagles players, each with their position, years played, and a short one-line blurb about why they matter. When I run it, let me type a search word and print any matching players plus their blurb. Add a "surprise me" option that shows a random one.

Steps

  1. Paste the brief above into Claude Code inside your project folder.
  2. Review the plan it proposes, then approve.
  3. Run it the way Claude tells you to (something like python card_finder.py or node card_finder.js).
  4. Test it: search "Kelce", search "Hurts", try the surprise-me option.
  5. Now ask for one change yourself — "add a rarity tier like a real trading card" or "add color to the output." This back-and-forth is the real skill.
  6. Commit your working version.

Checkpoint

If something breaks

Not sure if Python's installed? Just ask Claude: "do I have Python installed, and if not, walk me through installing it on Windows." It will check for you and guide you through it.
HOUR 5 Kickoff: Make It a Web Page & Put It Online
Windows +

Goal

Wrap Hour 4's tool in a simple web page, then publish it for free so a friend can use it from a link, no install required.

Steps

  1. Ask Claude: "turn this into a simple HTML page with a search box and button, using the same list of players, so I can open it in my browser."
  2. Open the new index.html by double-clicking it in File Explorer.
  3. Make a free account at github.com if you don't have one.
  4. Ask Claude: "help me create a new GitHub repo for this project and push my code."
  5. Turn on the free hosting: in the repo on github.com, go to Settings > Pages and set it to deploy from the main branch.
  6. Send the live link to a friend (or your wife) and watch them use it on their own computer or phone.

Checkpoint

If something breaks

No gh command? Download the GitHub CLI from cli.github.com, or just create the repo by hand on github.com and drag your files into it in the browser.
BONUS Zwei Sprachen: Teach Your Tool to Speak German
Windows +

Goal

Add an English / German toggle to your own Card Vault Finder, so your wife can actually use it too. This is exactly the feature powering the little "EN / DE" switch on the Play section below — a genuinely useful real-world skill: real apps ship in more than one language.

Steps

  1. Ask Claude directly: "add an EN / DE toggle to my finder — when someone clicks DE, translate the search placeholder, the filter buttons, and the 'no results' message into German, and switch back cleanly when they click EN."
  2. You don't need to know German yourself to do this — Claude does, and it'll ask you to confirm the exact wording if you want it phrased a certain way.
  3. Fun fact you'll notice along the way: German NFL fans mostly don't translate the football vocabulary — "Touchdown," "Quarterback," and "Linebacker" are just the same words in German. So player names, positions, and card blurbs stay in English even in the DE version; only the site's own buttons and messages change.
  4. Test both directions: click DE, confirm the search box, filter chips, and empty-state message all switched. Click back to EN and make sure nothing is left half-translated.
  5. Stretch goal: ask Claude to remember the last language picked using localStorage, so the page opens in German automatically next time — a nice small touch for someone who isn't you opening the link.

Checkpoint

If something breaks

If only part of the page translates, don't guess — tell Claude exactly which piece got missed, like "the shuffle button text didn't change when I clicked DE," and let it fix just that one spot.
BONUS The Trade Block: List Your Cards on eBay
Windows + iOS +

Goal

Use everything from Hours 1-5 to help catalog and sell your trading card collection on eBay, safely and for real money.

Read this first — the one rule that matters most

Condition is everything in this hobby. Corners, edges, centering, and surface are what buyers and graders actually look at — don't sleeve-flip, bend, or "just wipe off" a card before photographing it. If a card might be worth grading (PSA, BGS, SGC), photograph it as-is and research that before you touch it at all.

Steps

  1. If you kept a running list of cards in the Claude iPhone app back in Hour 1, pull it up now — that's your starting checklist.
  2. Ask Claude Code to help you build a simple tracker (a spreadsheet-style CSV file works great) with columns: player, year/set, card number, condition notes, estimated value, graded?, photographed?, listed?
  3. Photograph each card front and back in good, even light — no glare, no shadows, a plain background. A close-up of any corner or edge damage matters more than a wide shot.
  4. Research real comps — on eBay, filter for sold/completed listings of the exact same player, year, set, and card number. Asking prices lie; sold prices don't.
  5. Ask Claude to draft listing titles and descriptions from your tracker: "write me an eBay title and description for this card — include player, year, set, card number, condition, and the keywords collectors search for."
  6. NFL fandom has grown fast in Germany and across Europe, and Eagles cards get real bids from overseas collectors. If you want extra reach, ask Claude to draft a short German translation of your title and description to include alongside the English listing — the same translation skill from the Zwei Sprachen bonus above, just pointed at eBay instead of your own tool.
  7. Commons sell better bundled into small lots (5 or fewer). Save true rarities and graded slabs as solo listings; those are worth real money on their own.
  8. List a small test batch first — 5 to 10 cards — before doing the whole collection, so you can see what actually sells before pricing the rest.

Checkpoint

If something breaks

Not sure if a card is worth grading? Ask Claude to help you research the exact player, year, and set, then cross-check it against sold graded listings before you spend money on grading fees. And if you get stuck — ask Jason. He already runs a card & collectibles eBay store and does this exact workflow (photograph, research comps, bundle the commons into lots, sell the rare ones solo) every week.

The Cheat Sheet

Keep this open in a second tab during Hours 1-5.

Claude on iPhone

Great for photographing cards, dictating notes by voice, and drafting quick descriptions on the go. It can't run your project or terminal commands — that part stays on the Windows laptop.

Setup commands (Windows)

node -v
npm -v
npm install -g @anthropic-ai/claude-code
claude
claude doctor

Inside Claude Code

/help
/clear

Speak in plain English. Ask for a plan on anything big. Read the permission prompt before approving.

Windows Terminal navigation

dir
cd foldername
cd ..
mkdir foldername

Git (your undo button)

git init
git add .
git commit -m "message"
gh repo create

"'claude' is not recognized"

The terminal hasn't reloaded its PATH yet. Fully close Windows Terminal and reopen it. Still stuck? Restart the laptop once.

npm install fails

Try running Windows Terminal as Administrator (right-click > Run as administrator), then reinstall Node.js LTS from nodejs.org if that doesn't help.

"It keeps asking me to approve things"

That's working as intended — Claude Code asks before it changes files or runs commands. Read the prompt, then approve. It's a feature, not a bug.

Don't know what to type

Describe what you want in plain English, like texting a very capable, very literal friend. Specific beats vague.
